1) The system prompt enforces delegation1) O prompt de sistema impõe a delegação

The prompt is the contract: the orchestrator must route first, then synthesize.O prompt é o contrato: o orquestrador deve rotear primeiro, depois sintetizar.

ORCHESTRATOR_SYSTEM_PROMPT = (
    "You are an orchestrator agent. Your job is to route user requests to specialized sub-agents.

"
    "Available sub-agents (via delegate_to_agent):
"
    "- weather-agent: handles weather questions
"
    "- search-agent: handles general research/current topics

"
    "Rules:
"
    "1. Never answer weather or research questions directly.
"
    "2. Always delegate to one or more sub-agents first.
"
    "3. If the user asks both weather and research, call both agents.
"
    "4. Pass complete context to each sub-agent since they do not share memory with you.
"
    "5. After sub-agent results return, synthesize one final response for the user.
"
    "6. Before each delegation, include a brief planning sentence in the assistant content."
)
  • Rule 1 prevents the orchestrator from bypassing specialists.Regra 1 impede o orquestrador de ignorar especialistas.
  • Rule 4 matters because each delegation creates a fresh agent instance.Regra 4 importa porque cada delegação cria uma instância de agente nova.
  • Rule 5 makes the top-level answer a composition step, not a first-pass answer.Regra 5 torna a resposta do nível superior uma etapa de composição, não uma resposta imediata.
⚠️ Note⚠️ Nota

If the prompt merely said "use sub-agents when helpful," the model might answer directly. The explicit "never answer directly" rule is what turns this into a reliable pattern.Se o prompt apenas dissesse "use sub-agentes quando útil", o modelo poderia responder diretamente. A regra explícita "nunca responda diretamente" é o que torna isso um padrão confiável.

2) delegate_to_agent is a meta-tool2) delegate_to_agent é uma meta-ferramenta

In earlier lessons, tools called real-world functions such as search or fetch. Here, the tool creates and runs another Agent. That makes it a tool about tools and agents — a meta-tool.Nas lições anteriores, ferramentas chamavam funções do mundo real como busca ou obtenção. Aqui, a ferramenta cria e executa outro Agent. Isso a torna uma ferramenta sobre ferramentas e agentes — uma meta-ferramenta.

delegate_to_agent_tool_schema = {
    "type": "function",
    "function": {
        "name": "delegate_to_agent",
        "description": "Delegate a task to a specialized sub-agent and return its result.",
        "parameters": {
            "type": "object",
            "properties": {
                "agent_name": {
                    "type": "string",
                    "enum": ["weather-agent", "search-agent"],
                },
                "task_context": {
                    "type": "object",
                    "properties": {
                        "question": {"type": "string"}
                    },
                    "required": ["question"],
                },
            },
            "required": ["agent_name", "task_context"],
        },
    },
}
  • agent_name constrains routing to known specialists.agent_name limita o roteamento a especialistas conhecidos.
  • task_context forces the orchestrator to pass complete input, not just a label.task_context força o orquestrador a passar a entrada completa, não apenas um rótulo.
  • The return value comes back into the same core loop, so orchestration composes cleanly with the base agent design.O valor de retorno volta ao mesmo loop central, então a orquestração se combina limpar com o design base do agente.

3) The factory pattern closes over shared infrastructure3) O padrão de fábrica captura a infraestrutura compartilhada

make_delegate_tool() returns the actual callable tool. The outer function captures the client, trace flag, and discovered tool catalog.make_delegate_tool() retorna a ferramenta callable real. A função externa captura o cliente, o flag de rastreamento e o catálogo de ferramentas descoberto.

def make_delegate_tool(client, trace, tool_catalog):
    def delegate_to_agent(agent_name: str, task_context: Dict[str, Any]) -> Dict[str, Any]:
        question = str(task_context.get("question", "")).strip()
        if not question:
            return {"agent_name": agent_name, "error": "Missing task_context.question"}

        if agent_name == "weather-agent":
            sub_agent = build_weather_agent_with_tools(client, trace, tool_catalog)
        elif agent_name == "search-agent":
            sub_agent = build_search_agent_with_tools(client, trace, tool_catalog)
        else:
            return {"agent_name": agent_name, "error": f"Unknown agent_name '{agent_name}'"}

        answer = sub_agent.execute(question)
        return {"agent_name": agent_name, "question": question, "answer": answer}

    return delegate_to_agent
  • The outer function is a factory: it prepares runtime dependencies once.A função externa é uma fábrica: ela prepara as dependências de runtime uma vez.
  • The inner function is the tool handler the main agent can call repeatedly.A função interna é o manipulador de ferramenta que o agente principal pode chamar repetidamente.
  • Because sub_agent is created inside the tool call, every delegation starts with clean state.Como sub_agent é criado dentro da chamada de ferramenta, cada delegação começa com estado limpo.
Key InsightInsight Chave

This closure avoids global state. The orchestrator does not need a giant singleton registry; it simply captures the live client and tool catalog in the returned function.Este closure evita estado global. O orquestrador não precisa de um registro singleton gigante; ele simplesmente captura o cliente ativo e o catálogo de ferramentas na função retornada.

4) Sub-agent lifecycle: fresh instance, isolated context4) Ciclo de vida do sub-agente: instância nova, contexto isolado

Lifecycle per delegationCiclo de vida por delegação

  1. Receive agent_name and question.Receber agent_name e question.
  2. Build the specialist agent.Construir o agente especialista.
  3. Attach only that specialist's tools.Adicionar apenas as ferramentas daquele especialista.
  4. Run sub_agent.execute(question).Executar sub_agent.execute(question).
  5. Return structured results to the orchestrator.Retornar resultados estruturados ao orquestrador.

What isolation buys youO que o isolamento proporciona

  • No accidental cross-talk between agents.Sem interferência acidental entre agentes.
  • Different system prompts per role.Prompts de sistema diferentes por papel.
  • Different tool sets per role.Conjuntos de ferramentas diferentes por papel.
  • Easy reasoning about each specialist's behavior.Raciocínio fácil sobre o comportamento de cada especialista.

5) Dynamic tool discovery uses a plugin directory5) Descoberta dinâmica de ferramentas usa um diretório de plugins

Sub-agents do not hardcode raw function definitions. They load tools from the Sub-agentes não codificam definições de funções brutas. Eles carregam ferramentas do diretório tools/ directory, where each plugin folder contains a schema and a handler., onde cada pasta de plugin contém um esquema e um manipulador.

def discover_tools(tools_root: str) -> Dict[str, Dict[str, Any]]:
    root = Path(tools_root)
    catalog = {}
    for tool_dir in sorted(root.glob("*/")):
        schema_path = tool_dir / "schema.json"
        handler_path = tool_dir / "handler.py"
        if not schema_path.exists() or not handler_path.exists():
            continue

        schema = json.loads(schema_path.read_text(encoding="utf-8"))
        tool_name = schema.get("function", {}).get("name", "").strip()
        module = _load_python_module(handler_path)
        run_func = getattr(module, "run", None)

        catalog[tool_name] = {
            "schema": schema,
            "func": run_func,
            "path": str(tool_dir),
        }
    return catalog
  • schema.json defines what the model sees.schema.json define o que o modelo vê.
  • handler.py defines what Python executes.handler.py define o que o Python executa.
  • The catalog becomes a reusable inventory that multiple agent builders can slice differently.O catálogo torna-se um inventário reutilizável que múltiplos construtores de agentes podem fatiar de formas diferentes.

6) Response synthesis happens after delegation6) A síntese de respostas acontece após a delegação

The orchestrator can call one sub-agent or multiple sub-agents. Each tool result comes back as structured data with agent_name, question, and answer. The main agent then writes one combined response for the user.O orquestrador pode chamar um ou múltiplos sub-agentes. Cada resultado de ferramenta retorna como dados estruturados com agent_name, question e answer. O agente principal então escreve uma resposta combinada para o usuário.
