1) Schemas define the extraction target1) Esquemas definem o alvo de extração

The built-in invoice schema is just JSON Schema-style structure. The agent can swap it for contact, meeting, or a user-provided schema file.O esquema de fatura embutido é apenas uma estrutura no estilo JSON Schema. O agente pode substituí-lo por contato, reunião ou um arquivo de esquema fornecido pelo usuário.

_INVOICE_SCHEMA = {
    "type": "object",
    "properties": {
        "vendor": {"type": ["string", "null"], "description": "Supplier/vendor name."},
        "invoice_number": {"type": ["string", "null"], "description": "Invoice/NF-e number."},
        "issue_date": {"type": ["string", "null"], "description": "Issue date in YYYY-MM-DD format."},
        "due_date": {"type": ["string", "null"], "description": "Payment due date in YYYY-MM-DD format."},
        "currency": {"type": "string", "enum": ["BRL", "USD", "EUR", "GBP"]},
        "subtotal": {"type": ["number", "null"]},
        "tax_amount": {"type": ["number", "null"]},
        "total": {"type": "number"},
        "items": {
            "type": "array",
            "items": {
                "type": "object",
                "properties": {
                    "description": {"type": "string"},
                    "quantity": {"type": "number"},
                    "unit_price": {"type": "number"},
                    "line_total": {"type": "number"},
                },
                "required": ["description", "quantity", "unit_price", "line_total"],
            },
        },
        "notes": {"type": ["string", "null"]},
        "confidence": {"type": "string", "enum": ["high", "medium", "low"]},
    },
    "required": ["total", "confidence"],
}
  • required fields guarantee a minimum useful payload.Os campos required garantem um payload mínimo útil.
  • null-friendly properties let the model admit missing data instead of hallucinating it.As propriedades compatíveis com null permitem que o modelo admita dados ausentes em vez de aluciná-los.
  • confidence is also structured, so downstream code can branch on it.confidence também é estruturado, então o código downstream pode ramificar com base nele.

2) Any schema can be wrapped as a tool2) Qualquer esquema pode ser encapsulado como ferramenta

The agent does not need a custom tool schema per document type. It uses one builder that turns arbitrary JSON schema into a callable function definition.O agente não precisa de um esquema de ferramenta personalizado por tipo de documento. Ele usa um builder que transforma qualquer esquema JSON em uma definição de função callable.

def _build_tool_schema(schema_name: str, properties: Dict) -> Dict:
    return {
        "type": "function",
        "function": {
            "name": f"extract_{schema_name}",
            "description": (
                f"Extract structured {schema_name} data from the provided text. "
                "Use null for fields not present in the text. Never invent values."
            ),
            "parameters": properties,
        },
    }
  • The schema itself becomes the parameters object the model must satisfy.O próprio esquema se torna o objeto parameters que o modelo deve satisfazer.
  • The function name is dynamic, which keeps prompts readable: extract_invoice, extract_meeting, and so on.O nome da função é dinâmico, o que mantém os prompts legíveis: extract_invoice, extract_meeting, e assim por diante.
  • This is what makes the example extensible without touching the base Agent class.Isso é o que torna o exemplo extensível sem tocar na classe base Agent.
Key InsightInsight Chave

Once you understand this pattern, "structured extraction" is just "tool calling with a schema-shaped output contract."Uma vez que você entende este padrão, "extração estruturada" é apenas "chamada de ferramenta com um contrato de saída em formato de esquema".

3) The handler captures extracted fields in a global result3) O manipulador captura campos extraídos em um resultado global

The handler is intentionally simple. It validates and stores whatever arguments the model supplied, then reports lightweight status back to the loop.O manipulador é intencionalmente simples. Ele valida e armazena quaisquer argumentos que o modelo forneceu, então reporta um status leve de volta ao loop.

_extracted_result = None


def _make_extract_fn(schema_name: str, schema_props: Dict):
    def extract(**kwargs) -> Dict[str, Any]:
        global _extracted_result
        validated = {}
        props = schema_props.get("properties", {})
        for key, value in kwargs.items():
            if key in props:
                if value == "" or value == "N/A" or value == "n/a":
                    validated[key] = None
                else:
                    validated[key] = value
            else:
                validated[key] = value
        _extracted_result = validated
        return {
            "status": "extracted",
            "fields_filled": len([v for v in validated.values() if v is not None])
        }
    extract.__name__ = f"extract_{schema_name}"
    return extract
  • The closure remembers which schema it belongs to.O closure lembra a qual esquema pertence.
  • The global _extracted_result is crude but effective for a single-run CLI example.O global _extracted_result é rudimentar, mas eficaz para um exemplo de CLI de execução única.
  • The function returns a small status object because the real artifact is the captured payload, not the tool's textual return.A função retorna um pequeno objeto de status porque o artefato real é o payload capturado, não o retorno textual da ferramenta.
⚠️ Note⚠️ Nota

In production you would likely return the extracted data explicitly or pass a result object around. The global keeps the tutorial easy to follow because the control flow stays linear in main().Em produção você provavelmente retornaria os dados extraídos explicitamente ou passaria um objeto de resultado. O global mantém o tutorial fácil de seguir porque o fluxo de controle permanece linear em main().

4) The escape hatch handles mismatched documents4) A saída de emergência trata documentos incompatíveis

If the user asks for invoice extraction but provides a completely different document, the model has a clean non-success path.Se o usuário solicita extração de fatura mas fornece um documento completamente diferente, o modelo tem um caminho limpo de não-sucesso.

def _build_validate_tool() -> Dict:
    return {
        "type": "function",
        "function": {
            "name": "report_unrecognized",
            "description": "Call this when the text does not match the expected document type.",
            "parameters": {
                "type": "object",
                "properties": {
                    "reason": {"type": "string"}
                },
                "required": ["reason"],
            },
        },
    }


def _report_unrecognized(reason: str) -> Dict[str, Any]:
    global _extracted_result
    _extracted_result = {"error": "unrecognized", "reason": reason, "confidence": "low"}
    return {"status": "reported"}
  • This avoids forcing bad data into the extraction schema.Isso evita forçar dados ruins no esquema de extração.
  • The returned payload still has structure, including a low-confidence marker.O payload retornado ainda tem estrutura, incluindo um marcador de baixa confiança.
  • Fallback tools are often the difference between brittle demos and robust workflows.Ferramentas de fallback são frequentemente a diferença entre demos frágeis e fluxos de trabalho robustos.

5) The system prompt forces tool usage and self-reported confidence5) O prompt de sistema força o uso de ferramentas e confiança auto-relatada

def _build_system_prompt(schema_name: str) -> str:
    return f"""You are a document data extraction specialist.

Your task: extract structured {schema_name} data from the text provided by the user.

RULES:
1. ALWAYS call the extract_{schema_name} tool. Never respond with free text.
2. Use null for any field not present in the text. Do NOT invent or guess values.
3. Normalize dates to YYYY-MM-DD format.
4. Normalize monetary values to plain numbers.
5. If the document clearly does not match the expected type, call report_unrecognized instead.
6. Set confidence to "low" if you had to make significant assumptions.
7. Set confidence to "high" only if all required fields were clearly present.
"""
  • Rule 1 is the anchor: no free-text answer is acceptable.Regra 1 é a âncora: nenhuma resposta em texto livre é aceitável.
  • Rules 2-5 push the model toward honest, normalized output.Regras 2-5 levam o modelo a uma saída honesta e normalizada.
  • Rules 6-7 make confidence scoring part of the schema contract rather than a vague suggestion.Regras 6-7 tornam a pontuação de confiança parte do contrato do esquema, e não uma sugestão vaga.
Key InsightInsight Chave

The confidence field is produced by the model itself. That is not a guarantee of correctness, but it is a useful signal for downstream review, thresholds, or human-in-the-loop workflows.O campo de confiança é produzido pelo próprio modelo. Isso não é uma garantia de correção, mas é um sinal útil para revisão downstream, limites ou fluxos de trabalho com humano no loop.
