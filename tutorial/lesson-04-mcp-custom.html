Bootstrapping the server with FastMCPInicializando o servidor com FastMCP

The server starts with a tiny amount of framework code. FastMCP provides the runtime, and decorators register Python functions as tools.O servidor começa com uma quantidade pequena de código de framework. FastMCP fornece o runtime, e decorators registram funções Python como tools.

from mcp.server.fastmcp import FastMCP

mcp = FastMCP("Inventory", json_response=True)

@mcp.tool()
def query_products(...):
    ...

@mcp.tool()
def get_stats() -> dict:
    ...

if __name__ == "__main__":
    mcp.run(transport="stdio")

json_response=True is convenient because it encourages each tool to return structured data that the agent can reason over. The stdio transport keeps the server compatible with the client pattern from Lesson 3.json_response=True é conveniente porque incentiva cada tool a retornar dados estruturados sobre os quais o agent consegue raciocinar. O transporte stdio mantém o servidor compatível com o padrão de cliente da Lição 3.

SQLite as the backing storeSQLite como camada de armazenamento

The file auto-creates a local database and seeds it with sample products the first time it runs. That keeps the example reproducible: you launch the server and already have data to query.O arquivo cria automaticamente um banco local e o preenche com produtos de exemplo na primeira execução. Isso deixa o exemplo reproduzível: você sobe o servidor e já tem dados para consultar.

DB_PATH = Path(__file__).parent / "inventory.db"

def _get_db() -> sqlite3.Connection:
    conn = sqlite3.connect(str(DB_PATH))
    conn.row_factory = sqlite3.Row
    return conn

def _init_db() -> None:
    if DB_PATH.exists():
        return
    conn = _get_db()
    conn.executescript("""
        CREATE TABLE products (...);
        INSERT INTO products (name, category, price, stock, description) VALUES
            ('Wireless Mouse', 'Electronics', 29.99, 150, 'Ergonomic wireless mouse'),
            ('Standing Desk', 'Furniture', 399.99, 30, 'Electric height-adjustable desk');
    """)

For teaching, SQLite is perfect: no external dependency, real SQL, and persistent enough to demonstrate both reads and writes.Para ensinar, SQLite é perfeito: sem dependência externa, SQL real e persistente o bastante para demonstrar leituras e escritas.

Dynamic SQL in query_products()SQL dinâmico em query_products()

The query tool is a good example of parameterized, composable filters. It starts with a base query and adds clauses only when corresponding arguments are provided.A tool de query é um bom exemplo de filtros parametrizados e componíveis. Ela começa com uma consulta base e adiciona cláusulas só quando os argumentos correspondentes são fornecidos.

@mcp.tool()
def query_products(search=None, category=None, min_price=None, max_price=None, low_stock_only=False):
    conn = _get_db()
    query = "SELECT * FROM products WHERE 1=1"
    params: list = []

    if search:
        query += " AND (LOWER(name) LIKE ? OR LOWER(description) LIKE ?)"
        params.extend([f"%{search.lower()}%", f"%{search.lower()}%"])
    if category:
        query += " AND LOWER(category) = ?"
        params.append(category.lower())
    if min_price is not None:
        query += " AND price >= ?"
        params.append(min_price)
    if max_price is not None:
        query += " AND price <= ?"
        params.append(max_price)
    if low_stock_only:
        query += " AND stock < 50"

    rows = conn.execute(query + " ORDER BY name", params).fetchall()
    return [dict(row) for row in rows]

Because the tool exposes well-named arguments, the LLM does not have to invent SQL itself. It only needs to map user intent into structured parameters.Como a tool expõe argumentos bem nomeados, o LLM não precisa inventar SQL por conta própria. Ele só precisa mapear a intenção do usuário em parâmetros estruturados.

Key Insight:Insight Chave: good MCP tools expose the right level of abstraction. Let the database layer own SQL; let the model choose filter arguments.boas tools MCP expõem o nível certo de abstração. Deixe a camada de banco ser dona do SQL; deixe o modelo escolher os argumentos de filtro.

Aggregates with get_stats()Agregados com get_stats()

Agents often need summaries, not just rows. The stats tool pre-computes totals and grouped breakdowns so the model can explain the inventory instead of manually aggregating many query results.Agents muitas vezes precisam de resumos, não só linhas. A tool de stats pré-computa totais e agrupamentos para o modelo explicar o inventário em vez de agregar manualmente muitos resultados de query.

@mcp.tool()
def get_stats() -> dict:
    total = conn.execute("SELECT COUNT(*) as count, SUM(price * stock) as value FROM products").fetchone()
    categories = conn.execute(
        "SELECT category, COUNT(*) as count, SUM(stock) as total_stock FROM products GROUP BY category ORDER BY category"
    ).fetchall()
    low_stock = conn.execute(
        "SELECT name, stock, category FROM products WHERE stock < 50 ORDER BY stock"
    ).fetchall()
    return {
        "total_products": total["count"],
        "total_inventory_value": round(total["value"] or 0, 2),
        "categories": [dict(row) for row in categories],
        "low_stock_items": [dict(row) for row in low_stock],
    }

This is a strong pattern for business systems: push repetitive aggregation into tools so the model can spend its reasoning budget on explanation and decision support.Este é um padrão forte para sistemas de negócio: empurre agregações repetitivas para tools para o modelo gastar seu orçamento de raciocínio com explicação e apoio à decisão.

What's nextO que vem depois

Once you understand this server, you can extend it in several directions:Depois de entender este servidor, você pode estendê-lo em várias direções:

  • Add HTTP transport for remote access instead of local stdio.Adicionar transporte HTTP para acesso remoto em vez de stdio local.
  • Introduce authentication and authorization per tool or per server.Introduzir autenticação e autorização por tool ou por servidor.
  • Swap SQLite for Postgres, vector search, or a domain-specific API.Trocar SQLite por Postgres, vector search ou uma API específica de domínio.
  • Run multiple MCP servers and let one agent discover tools from all of them.Rodar vários servidores MCP e deixar um agent descobrir tools de todos eles.
  • Combine this with Lesson 6 guardrails or Lesson 7 approval gates for safer mutations.Combinar isso com os guardrails da Lição 6 ou os gates de aprovação da Lição 7 para mutações mais seguras.

The protocol stays the same even as the backend becomes more serious.O protocolo continua o mesmo mesmo quando o backend fica mais robusto.
