Skills vs. toolsHabilidades vs. ferramentas

A tool is one callable function schema. A skill is a package of intent: a name, a description, instructions, and a list of tools that should become available together.Uma ferramenta é um esquema de função callable. Uma habilidade é um pacote de intenção: um nome, uma descrição, instruções e uma lista de ferramentas que devem ficar disponíveis juntas.

Key InsightInsight Chave

Skills solve prompt bloat. Instead of exposing every tool and every instruction up front, the agent keeps a tiny catalog and loads the detailed instructions only when the task needs them.As habilidades resolvem o inchaço do prompt. Em vez de expor cada ferramenta e instrução de antemão, o agente mantém um catálogo mínimo e carrega as instruções detalhadas apenas quando a tarefa precisa.

1) The bootstrap tool is the only starting capability1) A ferramenta de bootstrap é a única capacidade inicial

SKILL.md is a human-readable manifestSKILL.md é um manifesto legível por humanos

Each skill lives in its own folder and describes itself in markdown. The parser reads structure from headings instead of requiring a separate JSON format.Cada habilidade vive em sua própria pasta e se descreve em markdown. O parser lê a estrutura a partir dos títulos em vez de exigir um formato JSON separado.

# search

## Description
Use this skill for general research, trends, and current-topic questions.

## Tools
- web_search
- web_fetch

## Instructions
- Run `web_search` before answering research/current-topic questions.
- Use `web_fetch` when snippets are insufficient.
def parse_skill_markdown(content: str) -> Dict[str, Any]:
    lines = [line.rstrip() for line in content.splitlines()]
    name = ""
    description_lines = []
    instructions_lines = []
    tools = []
    section = ""

    for line in lines:
        if line.startswith("# "):
            name = line[2:].strip().lower()
            continue
        if line.startswith("## "):
            section = line[3:].strip().lower()
            continue

        stripped = line.strip()
        if section == "description" and stripped:
            description_lines.append(stripped)
        elif section == "instructions" and stripped:
            instructions_lines.append(stripped)
        elif section == "tools" and stripped.startswith("- "):
            tools.append(stripped[2:].strip())

    return {
        "name": name,
        "description": " ".join(description_lines).strip(),
        "instructions": "
".join(instructions_lines).strip(),
        "tools": tools,
        "content": content,
    }
  • The parser returns both structured fields and the raw markdown content.O parser retorna tanto campos estruturados quanto o content markdown bruto.
  • The raw content matters later because the full skill text is injected into the live conversation.O conteúdo bruto importa mais tarde porque o texto completo da habilidade é injetado na conversa ativa.
  • Markdown keeps authoring friction low: adding a skill is mostly documentation plus tool names.O Markdown mantém o atrito de autoria baixo: adicionar uma habilidade é principalmente documentação mais nomes de ferramentas.

3) Catalog first, full instructions later3) Catálogo primeiro, instruções completas depois

The agent's base system prompt includes only a lightweight skill index. That is enough for the model to decide which skill to load, but not enough to bloat context with every detail.O prompt de sistema base do agente inclui apenas um índice leve de habilidades. Isso é suficiente para o modelo decidir qual habilidade carregar, mas não o suficiente para inflar o contexto com cada detalhe.

def build_skill_catalog_text(skill_catalog):
    lines = []
    for skill_name, item in sorted(skill_catalog.items()):
        desc = item.get("description", "")
        tools = ", ".join(item.get("tools", []))
        lines.append(f"- {skill_name}: {desc} (tools: {tools})")
    return "
".join(lines)

Skill catalogCatálogo de habilidades

Short index in the base prompt. Helps selection.Índice curto no prompt base. Auxilia na seleção.

Full skill contentConteúdo completo da habilidade

Injected only after loading. Helps execution.Injetado apenas após o carregamento. Auxilia na execução.

Key InsightInsight Chave

This mirrors retrieval patterns in larger systems: first retrieve a compact index, then pull detailed instructions only for the relevant item.Isso espelha padrões de recuperação em sistemas maiores: primeiro recuperar um índice compacto, depois buscar instruções detalhadas apenas para o item relevante.

4) The load function mutates the running agent4) A função de carregamento muta o agente em execução

This closure is the heart of the design. It updates the agent's messages, tools, and tool registry while the conversation is still running.Este closure é o coração do design. Ele atualiza as mensagens, ferramentas e registro de ferramentas do agente enquanto a conversa ainda está em execução.

def make_load_skill_tool(agent, skill_catalog, tool_catalog, trace):
    loaded_skills = set()
    activated_tools = set()

    def load_skill(skill_name: str) -> Dict[str, Any]:
        name = str(skill_name).strip().lower()
        skill = skill_catalog[name]

        if name not in loaded_skills:
            loaded_skills.add(name)
            agent.messages.append(
                {
                    "role": "system",
                    "content": (
                        f"[LOADED SKILL: {name}]
"
                        "Skill details below are now active:

"
                        f"{skill['content']}"
                    ),
                }
            )

        newly_activated = []
        for tool_name in skill.get("tools", []):
            if tool_name in activated_tools or tool_name not in tool_catalog:
                continue
            activated_tools.add(tool_name)
            newly_activated.append(tool_name)
            agent.tools.append(tool_catalog[tool_name]["schema"])
            agent.tool_registry[tool_name] = tool_catalog[tool_name]["func"]

        return {
            "ok": True,
            "skill_name": name,
            "newly_activated_tools": newly_activated,
            "active_tools": sorted(activated_tools),
        }

    return load_skill
  • agent.messages.append(...) injects the full skill instructions as a new system message.agent.messages.append(...) injeta as instruções completas da habilidade como uma nova mensagem de sistema.
  • agent.tools.append(...) changes what schemas the model can call on the next loop iteration.agent.tools.append(...) muda quais esquemas o modelo pode chamar na próxima iteração do loop.
  • agent.tool_registry[name] = ... wires the new schema name to executable Python.agent.tool_registry[name] = ... conecta o novo nome de esquema ao Python executável.
  • loaded_skills and activated_tools prevent duplicate injection.loaded_skills e activated_tools evitam injeção duplicada.
⚠️ Note⚠️ Nota

This example mutates the live agent object directly. That is simple and effective for a tutorial, but in a larger system you might wrap these mutations in helper methods for validation and observability.Este exemplo muta o objeto do agente ativo diretamente. Isso é simples e eficaz para um tutorial, mas em um sistema maior você pode encapsular essas mutações em métodos auxiliares para validação e observabilidade.

5) Tool discovery still comes from plugins5) A descoberta de ferramentas ainda vem de plugins

Skills do not contain Python code. They only reference tool names. The actual executable tools still come from the tools/ plugin catalog discovered at startup.As habilidades não contêm código Python. Elas apenas referenciam nomes de ferramentas. As ferramentas executáveis reais ainda vêm do catálogo de plugins tools/ descoberto na inicialização.

skills/skills/

  • What capability exists?Qual capacidade existe?
  • When should it be used?Quando deve ser usada?
  • Which tool names belong to it?Quais nomes de ferramentas pertencem a ela?

tools/tools/

  • What schema does the model see?Qual esquema o modelo vê?
  • What handler executes?Qual manipulador executa?
  • Optional tool-specific docs.Documentação opcional específica da ferramenta.

That separation is powerful: you can reorganize skills without rewriting handlers, or reuse the same tool in multiple skills.Essa separação é poderosa: você pode reorganizar habilidades sem reescrever manipuladores, ou reutilizar a mesma ferramenta em múltiplas habilidades.
