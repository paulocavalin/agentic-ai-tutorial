Why switch from stdio to HTTP?Por que mudar de stdio para HTTP?

With stdio, the agent process starts the MCP server and communicates through local pipes. That is simple and great for local demos, but it assumes both processes live on the same machine. HTTP breaks that coupling.Com stdio, o processo do agent inicia o servidor MCP e se comunica por pipes locais. Isso é simples e ótimo para demos locais, mas assume que ambos os processos vivem na mesma máquina. HTTP quebra esse acoplamento.

The server: FastMCP with Streamable HTTP transportO servidor: FastMCP com transporte Streamable HTTP

The file mcp_servers/notes_http_server.py defines a small notes service. The tools are ordinary Python functions decorated with @mcp.tool(), just like earlier lessons. The new part is the transport at startup.O arquivo mcp_servers/notes_http_server.py define um pequeno serviço de notas. As tools são funções Python comuns decoradas com @mcp.tool(), assim como nas lições anteriores. A parte nova é o transporte na inicialização.

from mcp.server.fastmcp import FastMCP

mcp = FastMCP("Notes Server")

@mcp.tool()
def create_note(title: str, content: str) -> str:
    """Create a new note. Returns the note ID."""
    notes = _load_notes()
    note_id = str(uuid.uuid4())[:8]
    notes[note_id] = {
        "title": title,
        "content": content,
        "created_at": datetime.now(timezone.utc).isoformat(),
    }
    _save_notes(notes)
    return f"Note created with ID: {note_id}"

# Start with HTTP transport
mcp.settings.port = 8000
mcp.run(transport="streamable-http")

Two details matter here. First, transport="streamable-http" tells FastMCP to expose the server over HTTP rather than stdio. Second, mcp.settings.port controls the listening port, which the example sets from CLI arguments before starting.Dois detalhes importam aqui. Primeiro, transport="streamable-http" diz ao FastMCP para expor o servidor por HTTP em vez de stdio. Segundo, mcp.settings.port controla a porta de escuta, que o exemplo define a partir de argumentos de linha de comando antes de iniciar.

NOTES_FILE = Path("notes.json")

def _load_notes() -> dict:
    if NOTES_FILE.exists():
        return json.loads(NOTES_FILE.read_text(encoding="utf-8"))
    return {}

def _save_notes(notes: dict) -> None:
    NOTES_FILE.write_text(json.dumps(notes, indent=2, ensure_ascii=False), encoding="utf-8")

This storage layer is intentionally simple. The important lesson is that transport and business logic are separate concerns: notes could later move to SQLite, Postgres, or a hosted database without changing how the agent calls the tools.Esta camada de armazenamento é intencionalmente simples. A lição importante é que transporte e lógica de negócio são preocupações separadas: as notas poderiam depois migrar para SQLite, Postgres ou um banco hospedado sem mudar a forma como o agent chama as tools.

The extended MCPAgentClientO MCPAgentClient estendido

From the agent's point of view, the biggest API change is tiny: lessons 3 and 4 passed StdioServerParameters, while this lesson passes a URL string. The client now accepts both.Do ponto de vista do agent, a maior mudança de API é mínima: as lições 3 e 4 passavam StdioServerParameters, enquanto esta lição passa uma string de URL. O cliente agora aceita ambos.

# Stdio (lessons 3-4)
async with MCPAgentClient(StdioServerParameters(command="...")) as mcp:

# HTTP (this lesson)
async with MCPAgentClient("http://localhost:8000/mcp") as mcp:

class MCPAgentClient:
    def __init__(self, server_params: Union[StdioServerParameters, str]) -> None:
        self.server_params = server_params

    @property
    def _is_http(self) -> bool:
        return isinstance(self.server_params, str)

    async def __aenter__(self):
        if self._is_http:
            self._transport_context = streamablehttp_client(self.server_params)
        else:
            self._transport_context = stdio_client(self.server_params)
        # ... rest is identical

Internally, the HTTP branch uses streamablehttp_client from the MCP SDK. After that branch point, initialization, tool discovery, session setup, and adapter creation are the same as before.Internamente, o branch HTTP usa streamablehttp_client do SDK de MCP. Depois desse ponto de decisão, inicialização, discovery de tools, setup de sessão e criação de adapters são iguais aos de antes.

Key Insight:Insight Chave: This is good client design: transport selection happens in one place, while the rest of the agent integration remains transport-agnostic.Isto é um bom design de cliente: a seleção do transporte acontece em um único lugar, enquanto o resto da integração do agent permanece agnóstico ao transporte.

The agent is almost unchangedO agent está quase inalterado

The file examples/mcp_http_agent.py looks very close to the Lesson 4 agent. It still creates an LLM client, opens an MCP context, converts discovered tools, builds an Agent, and runs the user prompt in a worker thread.O arquivo examples/mcp_http_agent.py parece muito com o agent da Lição 4. Ele ainda cria um cliente de LLM, abre um contexto MCP, converte as tools descobertas, monta um Agent e executa o prompt do usuário em uma thread de trabalho.

from core import Agent, MCPAgentClient, OllamaClient

async def main():
    client = OllamaClient(model="gemma4:12b")

    # Connect via HTTP — just pass the URL
    async with MCPAgentClient("http://localhost:8000/mcp") as mcp:
        loop = asyncio.get_running_loop()
        tools, registry = mcp.get_tools(loop)

        agent = Agent(client=client, tools=tools, tool_registry=registry, system=SYSTEM_PROMPT)
        result = await asyncio.to_thread(agent.execute, prompt)

That similarity is the teaching goal. When tool access is abstracted behind MCP, your agent logic does not care whether the server lives in the same process tree or across the network.Essa semelhança é o objetivo pedagógico. Quando o acesso às tools é abstraído por MCP, sua lógica de agent não se importa se o servidor vive na mesma árvore de processos ou do outro lado da rede.

Key takeawaysPrincipais aprendizados

  • HTTP transport lets MCP servers run remotely and serve more than one client.O transporte HTTP permite que servidores MCP rodem remotamente e atendam mais de um cliente.
  • The notes server keeps ordinary tool logic: CRUD functions plus a persistence layer.O servidor de notas mantém lógica de tool comum: funções CRUD mais uma camada de persistência.
  • MCPAgentClient changes only at the transport boundary, using streamablehttp_client for URLs.MCPAgentClient muda apenas na fronteira do transporte, usando streamablehttp_client para URLs.
  • Tool logic is decoupled from transport, so the same agent pattern works across stdio and HTTP.A lógica de tool é desacoplada do transporte, então o mesmo padrão de agent funciona em stdio e HTTP.
Final Insight:Insight Final: MCP becomes much more powerful once you stop thinking of tools as local subprocesses. Over HTTP, they can become reusable network services in a larger agent architecture.MCP fica muito mais poderoso quando você deixa de pensar em tools como subprocessos locais. Sobre HTTP, elas podem virar serviços de rede reutilizáveis em uma arquitetura maior de agents.
