The context window problemO problema da janela de contexto

Large language models only see the tokens you send on each call. If conversation history grows without bound, prompts become expensive, noisy, and easier for the model to forget the most relevant facts.Large language models só enxergam os tokens que você envia em cada chamada. Se o histórico cresce sem limite, os prompts ficam caros, ruidosos e facilitam que o modelo esqueça os fatos mais relevantes.

Key InsightInsight Chave

Memory management is not just about recall. It is also about prompt hygiene: keep the context small enough that the model can still focus.Gestão de memória não é só sobre lembrar. Também é higiene de prompt: mantenha o contexto pequeno o suficiente para o modelo continuar focado.

Failure modeModo problemático

Append-only history keeps everything, even details that no longer matter.Um histórico só de append guarda tudo, até detalhes que já não importam.

Managed modeModo gerenciado

Compress old turns into summary text and preserve only durable facts in a memory store.Comprima turnos antigos em um resumo e preserve só fatos duráveis em uma memória.

1) NaiveAgent: the simplest possible approach1) NaiveAgent: a abordagem mais simples possível

This class is intentionally small so the trade-off is obvious.Esta classe é pequena de propósito para deixar o trade-off óbvio.

NAIVE_SYSTEM = (
    "You are a helpful assistant. "
    "Answer questions clearly and concisely."
)

@dataclass
class NaiveAgent:
    model: str
    history: List[Dict] = field(default_factory=list)

    def chat(self, user_input: str) -> str:
        self.history.append({"role": "user", "content": user_input})
        messages = [{"role": "system", "content": NAIVE_SYSTEM}] + self.history
        response = _chat(messages, self.model)
        self.history.append({"role": "assistant", "content": response})
        return response

    @property
    def context_messages(self) -> int:
        return len(self.history)
  • Every turn appends two more messages: user + assistant.Cada turno adiciona mais duas mensagens: usuário + assistant.
  • Nothing is summarized, discarded, or extracted into durable memory.Nada é resumido, descartado ou extraído para uma memória durável.
  • It is easy to understand and easy to outgrow.É fácil de entender e fácil de ficar pequeno demais.

2) ManagedAgent adds two compression layers2) ManagedAgent adiciona duas camadas de compressão

@dataclass
class ManagedAgent:
    model: str
    session_name: str = "default"
    summary: str = ""
    recent: List[Dict] = field(default_factory=list)
    memory_store: Dict[str, str] = field(default_factory=dict)

    def chat(self, user_input: str) -> str:
        memory_block = ""
        if self.memory_store:
            items = "\n".join(f"  {k}: {v}" for k, v in self.memory_store.items())
            memory_block = f"\n\n[Long-term memory]\n{items}"

        system_content = MANAGED_SYSTEM + memory_block
        if self.summary:
            system_content += f"\n\n[Conversation summary so far]\n{self.summary}"

        messages = [{"role": "system", "content": system_content}]
        messages += self.recent
        messages.append({"role": "user", "content": user_input})

        response = _chat(messages, self.model)
        for mem in _extract_memories(response):
            self.remember(mem["key"], mem["value"])
        self.recent.append({"role": "user", "content": user_input})
        self.recent.append({"role": "assistant", "content": response})
        self._maybe_summarise()
        self._save()
        return response
  • summary stores compressed history.summary guarda o histórico comprimido.
  • recent keeps only the latest full-fidelity turns.recent mantém só os turnos mais recentes em fidelidade completa.
  • memory_store preserves named facts such as the user's city or job title.memory_store preserva fatos nomeados, como a cidade do usuário ou seu cargo.
  • _save() persists state after every response._save() persiste o estado depois de cada resposta.
Key InsightInsight Chave

Managed context is not one trick. It is the combination of short-term memory (recent), compressed memory (summary), and durable memory (memory_store).Contexto gerenciado não é um truque só. É a combinação de memória de curto prazo (recent), memória comprimida (summary) e memória durável (memory_store).

3) Summarization kicks in after a threshold3) A sumarização entra em ação depois de um limite

The file defines SUMMARY_THRESHOLD = 6 and SUMMARY_KEEP = 2. Once enough recent messages accumulate, the agent compresses older ones and keeps only the most recent pair(s) in full.O arquivo define SUMMARY_THRESHOLD = 6 e SUMMARY_KEEP = 2. Quando mensagens recentes suficientes se acumulam, o agent comprime as mais antigas e mantém só o(s) par(es) mais recente(s) por completo.

SUMMARISE_SYSTEM = (
    "You are a conversation summariser. "
    "Create a concise but complete summary of the conversation below. "
    "Preserve all factual details about the user (name, job, goals, preferences). "
    "Output only the summary text, no preamble."
)


def _maybe_summarise(self) -> None:
    if len(self.recent) < SUMMARY_THRESHOLD:
        return
    to_compress = self.recent[:-SUMMARY_KEEP]
    keep = self.recent[-SUMMARY_KEEP:]

    history_text = "\n".join(
        f"{m['role'].upper()}: {m['content']}" for m in to_compress
    )
    prefix = f"Existing summary:\n{self.summary}\n\n" if self.summary else ""
    self.summary = _chat(
        [
            {"role": "system", "content": SUMMARISE_SYSTEM},
            {"role": "user", "content": f"{prefix}New conversation:\n{history_text}"},
        ],
        self.model,
    )
    self.recent = keep
  • The threshold prevents summarizing on every single turn.O limite evita resumir a cada turno.
  • The existing summary is fed back in, so summarization is incremental.O resumo existente volta para a entrada, então a sumarização é incremental.
  • Keeping the newest messages verbatim preserves local conversational nuance.Manter as mensagens mais novas literalmente preserva nuances locais da conversa.

The code makes two LLM calls in this method, but only the second one is assigned to self.summary. That is fine for the tutorial's explanation: the important concept is the threshold-triggered compression step.O código faz duas chamadas de LLM neste método, mas só a segunda é atribuída a self.summary. Tudo bem para a explicação do tutorial: o conceito importante é a etapa de compressão acionada pelo limite.

4) Memory extraction is intentionally lightweight4) A extração de memória é propositalmente leve

The managed agent does not use tool calling here. Instead, it lets the assistant emit remember(key, value) text patterns and parses them with a regex.O agent gerenciado não usa tool calling aqui. Em vez disso, ele deixa o assistant emitir padrões de texto remember(key, value) e faz o parse com regex.

def _extract_memories(text: str) -> List[Dict[str, str]]:
    """Parse remember(key, value) calls from assistant response."""
    import re
    hits = re.findall(
        r'remember\(\s*["']?([^"'(),]+)["']?\s*,\s*["']?([^"'()]+)["']?\s*\)',
        text
    )
    return [{"key": k.strip(), "value": v.strip()} for k, v in hits]
  • This is less formal than tool calling, but very easy to demo.Isso é menos formal que tool calling, mas muito fácil de demonstrar.
  • The assistant can proactively mark useful facts during normal conversation.O assistant pode marcar fatos úteis proativamente durante a conversa normal.
  • After parsing, those facts are stored in a real Python dictionary for later turns.Depois do parse, esses fatos são guardados em um dicionário Python real para turnos futuros.

5) Session persistence makes memory survive process restarts5) A persistência de sessão faz a memória sobreviver a reinícios do processo

SESSIONS_DIR = Path.home() / ".ollama_memory_sessions"


def _save(self) -> None:
    data = {
        "summary": self.summary,
        "recent": self.recent,
        "memory_store": self.memory_store,
    }
    self._path.write_text(json.dumps(data, indent=2, ensure_ascii=False))


def _load(self) -> None:
    data = json.loads(self._path.read_text())
    self.summary = data.get("summary", "")
    self.recent = data.get("recent", [])
    self.memory_store = data.get("memory_store", {})
  • The session file is just JSON, which keeps the persistence story transparent.O arquivo de sessão é só JSON, o que deixa a persistência bem transparente.
  • Summary, recent messages, and memory store all survive between runs.Resumo, mensagens recentes e memória sobrevivem entre execuções.
  • This is enough to demonstrate long-term continuity without introducing a database.Isso já basta para demonstrar continuidade de longo prazo sem introduzir um banco de dados.
